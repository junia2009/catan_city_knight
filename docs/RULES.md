# 実装ルール仕様

本作に実装されている 3 モードのルールと、**公式ルールから簡略化・変更した点**をまとめる。
アプリ内の「📖 あそびかた」は要約版で、本書が正確な仕様。実装の対応箇所も併記する。

## 共通(全モード)

- 4人戦(人間1 + CPU 2〜3。CPU数の設定で3人戦も可)
- 盤面: 標準カタン(19ヘックス・数字トークン・港9つ)をシードからランダム生成
- 初期配置: スネーク順(1巡目 0→3、2巡目 3→0)。2軒目の隣接ヘックスから初期資源
- 7ロール: 手札が上限(基本7枚)超のプレイヤーは半分(切り捨て)を捨て、
  手番プレイヤーが盗賊を移動して隣接1人から1枚奪う
- 最長交易路(5本以上、+2点)。敵の建物で分断される。
  同数タイなら現保持者が維持、保持者が資格を失った同数タイは誰も持たない
- 銀行交易: 4:1、3:1港、2:1専用港
- プレイヤー間交易(拡張実装):
  - 人間 → CPU: 提案すると CPU が損得評価(難易度でマージンが変化)して応諾/拒否
  - CPU → 人間: 手番中に提案が届く(1手番1回まで。断られた CPU は4手番のあいだ
    その相手に再提案しない)
  - CPU ↔ CPU: 内部評価が折り合えば即時成立
- 勝利: 自分の手番中に規定点へ到達した瞬間(基本のみ手札の勝利点カード込みで判定)

## 基本モード(10点先取)

標準カタンそのまま:

- 発展カード25枚(騎士14・勝利点5・街道2・収穫2・独占2)。買ったターンは使えず、1手番1枚
- 最大騎士力(騎士3枚以上、+2点)

## 都市と騎士(13点先取)

実装: `src/rules/cak/` 一式。基本からの変更:

- **発展カード・最大騎士力は廃止**。代わりに進歩カード(下記)
- **商品**: 都市が森=布・山=コイン・畑=紙を産出(開拓地は資源のみ)。
  銀行在庫は各12枚。商品も手札上限に数え、捨て札の対象になる
- **初期配置は開拓地1+都市1**(2巡目が都市)
- **イベントダイス**: 毎ロールで振る。船(3面)=蛮族トラック前進、
  交易/政治/科学(各1面)=該当系統の改良レベルが 赤ダイス ≦ Lv+1 のプレイヤーに進歩カード
- **蛮族**: トラック7マス目で襲来。蛮族の強さ=全都市数、防衛力=活性騎士のレベル合計。
  - 防衛成功: 最大貢献者が「カタンの守護者」+1点(タイなら各自に進歩カード)
  - 防衛失敗: 最少貢献者(都市持ちのみ)の都市が開拓地に降格(メトロポリスは安全)
  - 襲来後はトラックがリセットされ、全騎士が不活性化
- **騎士**: 頂点に配置(羊1+鉱石1)。活性化(小麦1)。昇格(羊1+鉱石1、各レベル2体まで、
  Lv3は政治Lv3が必要)。活性騎士は次の手番から移動・盗賊追い払いができる。
  移動は自分の道網で到達できる頂点のみ、格下の敵騎士は追い出せる
- **城壁**: 都市の下にレンガ2で建設(3枚まで)。1枚につき手札上限+2。都市が降格すると失う
- **都市改良**: 都市を持っていれば商品で系統レベルを購入(Lv n のコストは商品 n 枚)。
  - Lv3 特典(公式準拠): 交易=**商館**(全商品が銀行と2:1)/ 政治=**要塞**(騎士Lv3へ昇格可)/
    科学=**水道橋**(産出のなかったロールで好きな資源1枚。7と自分が何かもらった時は除く)
  - **メトロポリス**: 系統で最初に Lv4 到達で獲得(+2点)。Lv5 で追い越されると移動
- **進歩カード**: 全54枚・公式構成(下表)。手札上限4枚。勝利点カード(憲法・印刷機)は
  引いた瞬間に公開・得点。買ったターンでも使える(獲得手段が配布のみのため)

### 進歩カード構成(公式と同一の54枚)

| 系統 | カード(枚数) |
|---|---|
| 交易(黄)18枚 | 商人(6) 資源独占(4) 商船隊(2) 交易独占(2) 豪商(2) 商業港(2) |
| 政治(青)18枚 | スパイ(3) 司教(2) 脱走兵(2) 外交官(2) 陰謀(2) 破壊工作員(2) 将軍(2) 王家の婚礼(2) 憲法(1) |
| 科学(緑)18枚 | 錬金術師(2) クレーン(2) 発明家(2) 灌漑(2) 医学(2) 鉱山(2) 街道建設(2) 鍛冶屋(2) 技師(1) 印刷機(1) |

各カードの効果はアプリ内「あそびかた → カード」に全掲載
(`src/rules/cak/progress-cards.js` の定義から自動生成)。

## 🐉 ドラゴンの島(12点先取・オリジナル)

実装: `src/rules/dragon.js`。基本モードがベース(発展カード・最大騎士力あり)で、
盗賊をドラゴンに置き換える:

- **ドラゴンの巣**: 最も出目の良い山ヘックス(なければ砂漠)。ドラゴン=盗賊コマとして
  産出を封鎖する。7ロールでの移動先も人間/CPUが選ぶ(通常の盗賊と同じ)
- **暴走**: ゾロ目(1/6の確率)が出ると、資源分配のあとドラゴンが
  「産出価値(pips × 建物重み、都市=2)最大のヘックス」へ飛び、**8手番のあいだ炎上**させる
  (産出停止+封鎖)。炎上中・現在地は次の対象から除外。対象がなければ巣へ帰る
- **略奪**: 炎上ヘックスに隣接する各プレイヤーは手札からランダム1枚を失う(銀行へ)
- **見張り塔**: 木1+レンガ1+鉱石1で自分の開拓地/都市の上に建設(1人2基まで)。
  塔が暴走ヘックスに隣接していれば略奪を防ぎ、**財宝**を獲得する
- **財宝**: 1個=+1勝利点+銀行からランダム資源1枚。ゲーム終了まで失われない
- ゾロ目は偶数なので7(捨て札・盗賊)と暴走は同時に起きない

## 公式ルールからの簡略化・変更点

CPU 戦のテンポと実装の単純さのため、以下は公式と異なる。
いずれも「選択をランダム/自動にする」方向の簡略化で、期待値はおおむね保たれる。

### カード効果の簡略化(都市と騎士)

| カード | 公式 | 本作 |
|---|---|---|
| 豪商 | 奪う側が資源を**見て選ぶ** | ランダムに2枚 |
| 司教 | 被害者の手札から引く(非公開ランダム)| 同じ(ランダム)だが対象は「隣接全員」から各1枚 |
| 王家の婚礼 | **渡す側が選んで**2枚渡す | 最も多く持っている札から自動で渡す |
| 商業港 | 相手が渡す商品を**選ぶ** | ランダムな商品1枚 |
| 脱走兵 | 相手が除去する騎士を**選ぶ**、配置先も自分で選ぶ | 相手の最弱騎士が自動選択、配置先も自動 |
| 鍛冶屋 | 昇格する騎士を**選ぶ** | 高レベル優先で自動選択(最大2体) |
| 陰謀 | 追い出された騎士の移動先を**持ち主が選ぶ** | 到達可能な空き頂点へ自動移動(なければ除去) |
| 外交官 | 相手の道の撤去 **または自分の道の移設** | 撤去のみ(移設は未実装) |
| スパイ | 相手の進歩カードを**見て選ぶ** | ランダムに1枚 |

### その他の簡略化

- **進歩カードの手札上限(4枚)超過**: 公式は「即座に1枚使うか捨てる」を選ぶが、
  本作は**新しいカードを引かず山札の底へ戻す**
- **盗賊・ドラゴンの略奪/暴走の被害**: 全てランダム抽選(公式も非公開ランダムなので実質同等)
- **交易の細部**: 公式にある「同じ手番での再交渉」の駆け引きはなく、
  人間→CPU は1提案ごとに即応答、CPU→人間 の提案は1手番1回まで
- **蛮族防衛のタイ処理**: 守護者タイの進歩カードは「最も育てている系統」の山札から自動で引く
  (公式はプレイヤーが系統を選ぶ)
- **商人**(進歩カード)は移動するまで盤上に残り保持者+1点(公式準拠)だが、
  移動の再配置は次に商人カードが使われた時のみ

### 実装上の同値な置き換え

- 山札のシャッフルや略奪の抽選はすべてシード付き乱数(`state.rng`)。
  公正性は `scripts/dice-audit.mjs` と `test/rng.test.js` の χ² 検定で監査済み
- ダイスは 2d6 を独立に生成(合計分布・ゾロ目率 1/6 を統計テストで確認)

# catan-web — カタン(都市と騎士対応予定)

Vanilla JavaScript / Canvas 2D で作るカタン。プレイヤー1人 vs CPU 2〜3体。
設計書に基づき、ルールエンジンと描画を完全分離したフェーズ開発を行う。

**現在: Phase 1(基本カタン + 貪欲法CPU)実装済み**

## 遊び方

ES Modules を使っているため、ローカルサーバー経由で開く:

```sh
npm run serve   # または python3 -m http.server 8000
# → http://localhost:8000/ を開く
```

- CPU数(2〜3体)とシードを指定して「新しいゲーム」
- 建設ボタン → 置ける場所がハイライト → クリックで選択 → 「確定」の2段階操作
- 交易・捨て札・略奪はダイアログで操作

## 開発

```sh
npm test              # ルールエンジンの単体テスト + セルフプレイ検証(Node のみ、ブラウザ不要)
npm run selfplay      # CPU 4体 × 200ゲームの自動対戦ゲート(引数でゲーム数変更可)
node scripts/selfplay.js 1000
```

## アーキテクチャ

```
src/
├── main.js            # 起動・ループ・入力モード管理(UIとCPUは同じ dispatch を使う)
├── state.js           # GameState 定義・初期化(JSON化可能な単一オブジェクト)
├── actions.js         # アクション定義と dispatch(validate → apply → postHooks)
├── rng.js             # シード制御の疑似乱数
├── rules/
│   ├── board.js       # 盤面生成・axial座標系・頂点/辺の正規化ID・隣接テーブル
│   ├── build.js       # 建設可否判定・コスト
│   ├── dice.js        # ダイス・資源分配(銀行在庫ルール含む)
│   ├── robber.js      # 盗賊・略奪
│   ├── trade.js       # 銀行/港交易(4:1 / 3:1 / 2:1)
│   └── victory.js     # 勝利点・最長交易路・最大騎士力
├── ai/
│   ├── legal-moves.js # 合法手列挙(validate の再利用)
│   ├── evaluator.js   # 盤面評価関数(産出期待値・多様性・港)
│   └── cpu-player.js  # 貪欲法 + ヒューリスティック思考ルーチン
├── render/
│   ├── board-render.js # Canvas 盤面描画(状態を描くだけ)
│   └── hud-render.js   # DOM HUD(手札・ボタン・ダイアログ・ログ)
└── input.js           # クリック → 頂点/辺/ヘックスの逆引き(閾値付き最近傍)
```

### 設計原則(設計書より)

1. ルールエンジンは純粋関数群。`canvas`/`ctx` を一切参照しない
2. 全ての状態変化は `dispatch(action)` の一本道(人間・CPU・UI共通)
3. 乱数はシード制御 — 同一シードは同一ゲーム(テストで決定性を保証)
4. 割り込みは `awaiting` 状態機械(捨て札・盗賊移動・初期配置)— Phase 2 の蛮族襲来等への布石

## Phase 1 実装済みルール

- 標準19ヘックス盤面のランダム生成(6/8非隣接保証)、港9箇所
- 初期配置(スネーク順、2軒目から初期資源)
- ダイス・資源分配(銀行在庫不足時の公式ルール)
- 建設(道・開拓地・都市、距離ルール、敵建物による道の分断、コマ数上限)
- 盗賊(7 → 8枚以上は半分捨て札 → 移動 → 略奪)
- 銀行/港交易(4:1、3:1港、2:1専用港)
- 発展カード(騎士・街道建設・収穫・独占・勝利点、購入ターン使用不可、1ターン1枚)
- 最長交易路・最大騎士力・勝利点10点判定

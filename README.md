# catan-web — カタン(都市と騎士)

Vanilla JavaScript / Canvas 2D で作るカタン。プレイヤー1人 vs CPU 2〜3体。
設計書に基づき、ルールエンジンと描画を完全分離したフェーズ開発を行う。

**現在: Phase 2(都市と騎士の盤面要素)まで実装済み** — 基本カタンと「都市と騎士」の両モードで遊べる。

## 遊び方

ES Modules を使っているため、ローカルサーバー経由で開く:

```sh
npm run serve   # または python3 -m http.server 8000
# → http://localhost:8000/ を開く
```

- ⚙️(設定)からモード・CPU数(2〜3体)・シードを指定して「新しいゲーム」
- **3D表示**(デフォルト): ドラッグで回転、ホイール/ピンチでズーム。設定で2Dに切替可
- 建設ボタン → 置ける場所がハイライト → クリックで選択 → 「確定」の2段階操作
- 交易・捨て札・略奪はダイアログで操作
- **スマホ対応**: 縦持ちでは盤面が全画面・縦構図になり、プレイヤーは上部チップ(タップで詳細)、
  ログは📜、設定は⚙️のボトムシートで操作
- **PWA対応**: ホーム画面に追加すればアプリとして起動できる。
  Service Worker はネットワーク優先(オンライン時は常に最新版を取得、
  キャッシュはオフライン時のフォールバック専用)なので、古いバージョンが残らない

## 開発

```sh
npm test              # ルールエンジンの単体テスト + セルフプレイ検証(Node のみ、ブラウザ不要)
npm run selfplay      # CPU 4体 × 200ゲームの自動対戦ゲート(引数でゲーム数変更可)
node scripts/selfplay.js 1000
```

## アーキテクチャ

```
src/
├── main.js            # 起動・ループ・入力モード管理(UIとCPUは同じ dispatch を使う)
├── state.js           # GameState 定義・初期化(JSON化可能な単一オブジェクト)
├── actions.js         # アクション定義と dispatch(validate → apply → postHooks)
├── rng.js             # シード制御の疑似乱数
├── rules/
│   ├── board.js       # 盤面生成・axial座標系・頂点/辺の正規化ID・隣接テーブル
│   ├── build.js       # 建設可否判定・コスト
│   ├── dice.js        # ダイス・資源分配(銀行在庫ルール含む)
│   ├── robber.js      # 盗賊・略奪
│   ├── trade.js       # 銀行/港交易(4:1 / 3:1 / 2:1)
│   └── victory.js     # 勝利点・最長交易路・最大騎士力
├── ai/
│   ├── legal-moves.js # 合法手列挙(validate の再利用)
│   ├── evaluator.js   # 盤面評価関数(産出期待値・多様性・港)
│   └── cpu-player.js  # 貪欲法 + ヒューリスティック思考ルーチン
├── render/
│   ├── board-render.js # Canvas 2D 盤面描画(状態を描くだけ)
│   └── hud-render.js   # DOM HUD(手札・ボタン・ダイアログ・ログ)
├── render3d/
│   └── board3d.js     # Three.js 3D 盤面描画(同じ GameState を描く別レンダラー)
└── input.js           # クリック → 頂点/辺/ヘックスの逆引き(2D: 最近傍 / 3D: レイキャスト)
```

3D 表示は `vendor/` にベンダリングした Three.js(MIT License)を importmap で読み込む。
ビルド工程は不要のまま。WebGL が使えない環境では自動で 2D にフォールバックする。

### 設計原則(設計書より)

1. ルールエンジンは純粋関数群。`canvas`/`ctx` を一切参照しない
2. 全ての状態変化は `dispatch(action)` の一本道(人間・CPU・UI共通)
3. 乱数はシード制御 — 同一シードは同一ゲーム(テストで決定性を保証)
4. 割り込みは `awaiting` 状態機械(捨て札・盗賊移動・初期配置)— Phase 2 の蛮族襲来等への布石

## Phase 1 実装済みルール(基本カタン)

- 標準19ヘックス盤面のランダム生成(6/8非隣接保証)、港9箇所
- 初期配置(スネーク順、2軒目から初期資源)
- ダイス・資源分配(銀行在庫不足時の公式ルール)
- 建設(道・開拓地・都市、距離ルール、敵建物による道の分断、コマ数上限)
- 盗賊(7 → 8枚以上は半分捨て札 → 移動 → 略奪)
- 銀行/港交易(4:1、3:1港、2:1専用港)
- 発展カード(騎士・街道建設・収穫・独占・勝利点、購入ターン使用不可、1ターン1枚)
- 最長交易路・最大騎士力・勝利点10点判定

## Phase 2 実装済みルール(都市と騎士)

- 勝利点13点、初期配置は開拓地×1+都市×1、発展カード・最大騎士力は廃止
- 赤+黄+イベントダイス。都市は商品の出る地形で資源1+商品1(森=紙、山=コイン、牧=布)
- 蛮族襲来: 船の目でトラック前進、7マス目で「蛮族の強さ(都市数) vs 防衛力(活性騎士Lv合計)」
  - 防衛成功 → 最大貢献者に守護者1点(同点は全員に進歩カード)
  - 防衛失敗 → 最少貢献者の都市が降格(メトロポリス除外)、襲来後は全騎士不活性
- 騎士: 建設(不活性)→ 活性化(小麦1)→ 昇格(Lv3は政治Lv3必須)。移動・追い出し・盗賊追い払い
- 都市改良: 3系統×Lv5(コスト=Lv枚の商品)、Lv4でメトロポリス(+2点、Lv5で移動)
  - Lv3の特殊能力(公式): 交易=商館(全商品2:1)、政治=要塞(騎士Lv3解禁)、科学=水道橋(収入0のとき資源1枚)
- 進歩カード: イベントダイスの色 + 赤ダイス ≦ Lv+1 で獲得。手札上限4枚、勝利点カードは公開即得点
- 城壁: レンガ2で都市の下に建設(最大3枚)、手札上限 7 → +2/枚。都市降格で城壁も除去

## Phase 3 実装済みルール(進歩カード全54枚 + CPU高度化)

- 公式構成の3系統×18枚(交易・政治・科学)。カード効果はプラグイン実装(`rules/cak/progress-cards.js`)
  - 交易: 商人(2:1交易+1点、盤面コマあり)・商船隊・資源独占・交易独占・豪商・商業港
  - 政治: 司教・憲法(VP)・脱走兵・外交官・陰謀・破壊工作員(捨て札割り込み再利用)・スパイ・将軍・王家の婚礼
  - 科学: 錬金術師(ロール前に出目指定)・クレーン・技師・発明家(数字トークン交換)・灌漑・医学・鉱山・印刷機(VP)・街道建設・鍛冶屋
  - 簡略化: 豪商/司教の略奪はランダム、婚礼/脱走兵の「相手が選ぶ」は自動選択、手札上限超過は山札の底へ
- CPU の使用判断はカード別評価プラグイン(`ai/progress-ai.js`)— 設計書 §7.4 の aiScore 方式
- 難易度(弱い/普通/強い): 評価関数への決定的ノイズ + カード使用閾値 + 交易受諾マージンで実現
